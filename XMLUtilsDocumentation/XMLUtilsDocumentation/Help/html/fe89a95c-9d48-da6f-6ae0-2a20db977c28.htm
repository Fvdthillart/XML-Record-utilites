Remarks

The anonymizer class is used to replace meaningful values with meaningless values. Anonymizing is done to cloak personal information like names, account number and anything that can be used to identify a person and use that information in a way that was not intended by the person, identified by the information. The AnonymizerWithRules class extends the Anonymizer class by providing functions to import a file with Anonymization rules. These rules provide a filemask and the data elements for instances of the AnonymizeRule class.

A rule file consists of the following elements, seperated by a ; :

  1. Filemask - Part of the name of the file to be anonymized. This way one rules file can apply to multiple files to be anonymized.
  2. Location - Indicates the element to be anonymized. For a CSV file this is a column name or index. For an XML file, this is an xpath expression
  3. FilterLocation -

    First part of a filter.Indicates the element whose value is to be filtered on. For CSV it can be a header name or index. For XML, it's an xpath expression that identifies 1 XML element

    Optional when none of the other parts of the filer are empty

  4. FilterOperator -

    Second part of the filter. Comparison operator for the filter

    Optional when none of the other parts of the filer are empty

  5. FilterValue -

    Third part of a filter. Contains the value to which the filter compares the data element, specified in FilterLocation

    Optional when none of the other parts of the filer are empty

Examples

Examples of anonymization rules

Example for CSV file (no header)

  • House;1;Houseaidvnr
  • test;1;Text

Example for an XML file

  • House;//hs:Rel_Match/hs:AchterNaam;Text
  • House;//hs:Rel_Match/hs:GeboorteDt;Date
